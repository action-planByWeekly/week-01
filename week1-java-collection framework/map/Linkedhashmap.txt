1. What is LinkedHashMap?

LinkedHashMap = HashMap + Doubly Linked List

Stores key-value pairs

Maintains insertion order (or access order, if configured)

Allows 1 null key, multiple null values

âœ… Key idea: Ordered HashMap

ðŸ”µ 2. Why LinkedHashMap Exists

HashMap is fast but unordered

Sometimes, we need predictable iteration order

LinkedHashMap solves this:

Iterates keys/values in order they were inserted

Can also maintain access order â†’ useful for LRU caches

-------------------------------------------------------------------------------------------------------
ðŸ”µ 3. Internal Structure

Internally, LinkedHashMap is:

HashMap (array of buckets) + Doubly Linked List (maintains order)


Each Node contains:

key

value

hash

next pointer (for bucket collisions)

before and after pointers â†’ doubly linked list

Bucket (array index) â†’ Node â†’ Node â†’ Node
Doubly Linked List: head â†” Node â†” Node â†” tail


Linked list keeps insertion order

Iteration uses linked list, not bucket array

-------------------------------------------------------------------------------------------------------
ðŸ”µ 4. How put() Works

Compute hash of key â†’ determine bucket

Check if key exists in bucket (using equals)

If key exists â†’ update value â†’ move node in linked list if access-order enabled

If key does not exist â†’ insert node at end of linked list and insert into bucket

Check load factor â†’ resize if needed

-------------------------------------------------------------------------------------------------------
ðŸ”µ 5. Access Order (Optional Feature)

Constructor allows accessOrder = true

Node is moved to tail whenever it is accessed

Useful for LRU Cache (Least Recently Used)

Example:

LinkedHashMap<String, Integer> map = new LinkedHashMap<>(16, 0.75f, true);


Accessing a key â†’ moves it to tail â†’ oldest entries remain at head

-------------------------------------------------------------------------------------------------------
ðŸ”µ 6. Time Complexity
Operation	Complexity
put()	O(1) average
get()	O(1) average
remove()	O(1) average
iteration	O(n)

Slightly slower than HashMap â†’ maintaining linked list pointers

-------------------------------------------------------------------------------------------------------

8. Advantages of LinkedHashMap

Maintains insertion order

Can maintain access order â†’ useful for caches

Fast operations (almost same as HashMap)

Predictable iteration
-------------------------------------------------------------------------------------------------------
ðŸ”µ 9. Disadvantages

Slightly more memory â†’ linked list pointers

Slightly slower than HashMap

Still not sorted â†’ use TreeMap for sorting

Example Code


import java.util.LinkedHashMap;

public class LinkedHashMapExample {
    public static void main(String[] args) {
        LinkedHashMap<String, Integer> map = new LinkedHashMap<>();

        map.put("Alice", 25);
        map.put("Bob", 30);
        map.put("Charlie", 20);

        System.out.println(map); // [Alice=25, Bob=30, Charlie=20] insertion order

        map.get("Alice"); // access
        for(String key : map.keySet()) {
            System.out.println(key + " -> " + map.get(key));
        }
    }
}
